# Vault Update — Earned Institutional Write

Direct vault writes alter institutional memory. Use this only for durable knowledge that future agents must find. If it is routine progress, tentative, or merely useful to the current task, write it to the session doc instead.

## Eligibility

Write directly only for:

- A confirmed correction to existing vault knowledge.
- A hard decision with trade-offs and rationale.
- A repeated pattern that will prevent future failures.
- An architectural truth discovered through implementation or verified operation.

If unsure, it is not earned. Put it in the session doc and let Administratum promote it later.

## Procedure

1. State what you will write and why it is earned.
2. Locate the target note. Read it before editing.
3. Follow the vault's folder, frontmatter, FileClass, and naming rules from local vault guidance.
4. Make the smallest correct write: append a section, patch stale text, or create a properly placed note.
5. Preserve provenance: include related session doc(s) in frontmatter where appropriate and add a short attribution/context line in the body.
6. Record the vault write in your session doc via `$session-update`.

## Quality Bar

- Do not overwrite another agent's live edits blindly.
- Do not create duplicate notes when an existing canonical note should be updated.
- Do not promote speculation to canon.
- MiniMax/Imperial Guard tier agents do not use this procedure; they write session docs only.

Legacy `$vault-canon` is the compatibility alias for this same earned-write procedure.
